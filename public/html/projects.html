Aptus Admin: a site management tool

Category: Tech
Date: May 2018

Aptus Admin is a tool for people managing sites for multiple clients. This being a collaborative project, I designed and developed the server side of it, focussing on scalability and security. Using Express and nodejs, I created all routes to and from the server. Since access to the site had to be restricted, I developed an authentication module from the ground up. The module includes JWT generation and validation and deals with cookie handling (sending and parsing) to store tokens. I also created a separate database module that deals with all the SQLite queries, making it easy for other developers to communicate to the database using set functions.

Spelling checker

Category: Tech
Date: Dec 2017

Using the same BST ADT that I creating for an earlier project, I wrote a spelling checker using binary search trees. This short program reads in a dictionary file and stores each word in a dictionary BST. It then reads in a second text file and checks each word of this file with the words stored in the dictionary BST. Each misspelt word is printed out and also added to a second (misspelt) BST so that no misspelled word is printed twice.

Visualising binary search trees

Category: Tech
Date: Dec 2017

I used this project to learn more about binary search trees. To start with, I implemented an Abstract Data Type (ADT) for binary search trees (BST). Using C as the language for this project, the ADT allows you store keys in an ordered manner and allows you to perform tasks such as finding the size of the tree, add entries to the tree and find a particular entry in the tree. For the visualisation part of the project I used Graphviz and the dot language. I implemented a dot ADT and a user file, that together create a dotfile of a string provided by the user. The generated dotfile is then rendered into a png, which visualises the tree. To play around with the functionalities of Graphviz a bit, I made each node blue when it contains the letter 'u'.

NAND-gates: JK and D-latch MS flip flops

Category: Tech
Date: Nov 2017

The aim of this project was to learn more about flip flop circuits. I build a positive triggered JK Master Slave Flip Flop and a D-type Master Slave flip flop. Just as with the full adder project, I used Logisim to design a simulation of the circuit and - once I knew it was working - built both the flip flops using these NAND boards. However, figuring out how to built these flip flops was a bit harder, as I couldn't proof these circuits with boolean logic (it's sequential logic instead). However, by starting small with first buildig a SR latch, then moving on towards SR FF, then the D-latch FF, the JK FF and only THEN adding the master slave components to it, definitely made it doable (also, truth tables are your best friend here).

Text comparison

Category: Tech
Date: Nov 2017

This program - written in C - compares two text documents with each other and tells you how many unique words are contained in each file as well as giving you the amount of words both files have in common. To make this work, I wrote an ADT for sets that contains operations such as creating sets, add elements to them, return the size, check a set for a certain value etc. I also used Valgrind to check the program for memory loss, to ensure there were no leaks in my program. A makefile is provided to simply start the program (the files that need to be compared with each other need to be provided by the user of the program).

NAND-boards: Full and Half Adder

Category: Tech
Date: Oct 2017

This project was my first experience with eletronics and circuit boards. The idea was to build a chained full adder that could perform additions up to a sum of N-bits (depending on the amount of adders in the chain). The way I approached designing this circuit, was by writing down its truth table first. With the help of the truth table, I could then point out the logical relationship in the form of formulas S (the Sum) and C (the carry). By simplifying the formulas, it shows you exactly how the logic gates in the circuit are connected with each other. I then created the circuits as simulations in Logisim to see if my design was correct. Once that was finished, I used these NAND boards to build the circuits 'for real'.
